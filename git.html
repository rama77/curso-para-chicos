Estados

  • working directory

    • Todos los archivos que estamos trabajando y que todavia no le informamos a git que van a ser parte del proyecto

  • staging area

    • En este estado estan los archivos que vamos a subir al proyecto

  • repository

    • Este es el proyecto con todos los archivos que forman parte de nuestro repositorio

Comandos básicos

Todos los comandos se ejecutan desde una terminal

  • Init

    • se utiliza para inicializar git en una proyecto. Hay que ir a la carpeta y ejecutar el comando git init

  • add [archivo]

    • permite subir archivos desde el working area al staging area

    y sirve para decir cuales archivos van a formar parte del proximo commit. Se utiliza ejecutando git add nombreArchivo.extension. Si se quiere subir todo lo pemdiente, se puede cambiar el nombre del archivo por un punto, asi:

    git add .

  • checkout

    • este comando sirve para varias cosas, queremos sacar un archivo del working area y recuperar la version anterior, hay que correr el comando git checkout nombre.archivo

    Tambien sirve para cambiar entre branchs (lo vemos despues)

  • diff
  • log
  • status
    • Este comando sirve para saber que archivos estan el el staging area y cuales no.
  • commit
    • con commit enviamos los archivos que estan en el working area, al repositorio. Siempre hay que informar un comentario para ir documentando lo que se va integrando al proyecto. Se ejecuta asi git commit -m "Aca va el comentario"
